# Contributing

Thanks for your interest in contributing to StackMCP.

StackMCP is an open-source project, and contributions of all kinds are welcome — documentation, bug reports, feature ideas, and code.

## How to contribute

### Reporting bugs

Open a GitHub issue with:

- A clear, descriptive title
- Steps to reproduce (what you did, what happened, what you expected)
- Your environment (OS, StackMCP version if known)

### Suggesting features

Open a GitHub issue describing:

- What you're trying to accomplish
- How StackMCP could help
- Any prior art or alternatives you've considered

### Documentation and examples

Documentation improvements, new examples, and workflow templates are especially welcome. See the [workflows README](workflows/README.md) for template structure.

### Code contributions

1. Fork the repository
2. Create a branch for your change
3. Make your changes
4. Open a pull request with a clear description of what it does and why

All contributions are subject to the [MIT License](LICENSE) that governs this project.

## Guidelines

- Keep documentation public-safe — no internal paths, secrets, or private architecture details
- Workflow templates should be self-contained and well-commented
- Examples should be honest about what's implemented vs. planned
